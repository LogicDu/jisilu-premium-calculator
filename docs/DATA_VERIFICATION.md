# 数据准确性验证方案

## 一、问题定位分析

### 1.1 数据流追踪

```
集思录页面(DOM) → 基金代码 → 天天基金API → 实时估值 → 溢价率计算 → 展示
                    ↓                           ↓
              场内价格(来自DOM)              (价格-估值)/估值
```

### 1.2 潜在错误点

| 环节 | 错误类型 | 现象 |
|------|----------|------|
| 基金代码提取 | 代码错误/截断 | 估值API返回空或错误基金 |
| 场内价格解析 | parseFloat失败 | 溢价率显示"-" |
| API请求 | 网络错误/超时 | 估值显示"-" |
| JSONP解析 | 格式变化 | 估值解析失败 |
| 缓存 | 数据过期 | 估值不是最新 |
| 计算精度 | 浮点误差 | 溢价率偏差 |

## 二、验证方案

### 方案一：添加调试面板（推荐）

在脚本中添加一个可开关的调试面板，显示原始数据和计算过程。

#### 实现步骤：

1. **添加调试面板开关**
   - 在表格上方添加"显示调试信息"按钮
   - 点击后展开调试面板

2. **调试面板内容**
   ```
   ┌─────────────────────────────────────────────────────────┐
   │ 🔍 数据验证面板                                           │
   ├─────────────────────────────────────────────────────────┤
   │ 基金代码: 161725                                         │
   │ 场内价格: 0.498 (来自DOM: "0.498")                       │
   │ 实时估值: 0.4865 (API返回: {"gsz": "0.4865", ...})       │
   │ 计算过程: (0.498 - 0.4865) / 0.4865 × 100 = 2.36%       │
   │ API响应时间: 234ms                                       │
   │ 数据时间: 2026-03-12 14:35:28                            │
   │                                                         │
   │ [验证链接] 天天基金: https://fund.eastmoney.com/161725  │
   │ [验证链接] 集思录: https://www.jisilu.cn/data/lof/...   │
   └─────────────────────────────────────────────────────────┘
   ```

3. **交互验证**
   - 点击基金代码 → 打开天天基金官方页面对比
   - 显示API原始响应（JSON格式）
   - 显示计算公式和中间值

### 方案四：自动化测试脚本

创建独立的测试脚本，模拟API调用和数据解析。

#### 测试用例：

```javascript
// 测试数据
const testCases = [
    { code: '161725', expectedPrice: 0.498, expectedEstimate: 0.4865 },
    { code: '162411', expectedPrice: 1.234, expectedEstimate: 1.2200 },
    // ...
];

// 自动验证
testCases.forEach(test => {
    const apiResult = await fetchEstimate(test.code);
    const premiumRate = calculatePremiumRate(test.expectedPrice, apiResult);
    console.log(`${test.code}: 估值=${apiResult}, 溢价率=${premiumRate}%`);
});
```

## 五、常见问题排查

### Q1: 估值显示"-"

**排查步骤：**
1. 检查控制台是否有API错误
2. 检查基金代码是否正确
3. 访问天天基金官网确认该基金是否有实时估值
4. 检查是否非交易时段

### Q2: 溢价率偏差大

**排查步骤：**
1. 确认场内价格是否最新（刷新页面）
2. 确认估值是否最新（检查缓存时间）
3. 对比其他平台计算结果
4. 检查是否为QDII基金（时区差异）

### Q3: 排序错乱

**排查步骤：**
1. 检查data-premium-rate属性值
2. 检查是否有null值干扰排序
3. 确认列索引是否正确

## 六、建议的数据展示格式

为了提高可信度，建议修改展示格式：

```
┌─────────────────────────────────────────┐
│ 溢价率: +2.36%                          │
│ ─────────────────────────────────────── │
│ 场内价: 0.498  实时估值: 0.4865         │
│ 更新: 14:35:28 (缓存: 23s)              │
└─────────────────────────────────────────┘
```

或者简化版：

```
溢价率: +2.36% ⏱14:35
```

这样可以：
1. 让用户看到数据时间
2. 了解数据新鲜度
3. 方便追溯验证
